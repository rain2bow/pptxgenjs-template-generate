# 新分支开发规划

## 优化目标
本次优化主要针对于本项目的不足, 针对速度,兼容,内容等 ; 依次执行以下目标, 执行完一个优化进行测试, 每次执行完必须commit再继续

### 1. 名称澄清
- layout 对于 纯文本/有图片/有图表 的, 在命名上进行区分避免误选, 例如 text-title, image-grid, chart-chart 等(仅作示例, 名称你可以自行调整)
- 不同style的layout名称统一 : 完全一致而不是兼容, 确保类型相同, 方便以后修改和新增模板
- layout内部的名称统一 : 例如某页, title底下的内容, 现在可能是sections, 可能是items, 这方面的命名也要尽量统一, 涉及的字段名越少越好

### 2. 流程优化
- 添加style选择阶段, 根据用户输入使用 askUserQuestion 给出当前的模板介绍让其选择
- 移除plan和字数md阶段, 选择完style后, 生成一份该style包含所有layout的json样例的md文件供模型参考选择, 不需要给出字数限制, 因为即使给出了ai也很难严格遵循

### 3. 兼容性优化
- 使所有 不含图/含图 的layout 都有一份对应的 含图/不含图 的layout备份 (例如text-grid <-> image-grid) 且其中字段兼容
- 完成上一项以后(必须要确定字段除了图片以外可以完全兼容), 将缺图/不能含图的报错改为提示其改为对应的layout (涉及图表的因为跟数据有关,不要修改)

### 4. 模板引入优化
- 新建一个新增模板的方案, 需要用户生成pptxgenjs js以后可以通过无损(即纯新增, 不用涉及到改动原有的js代码导致改错风险)方式, 新增style模板并使用
- 给出一个创建模板的指南md,方便别的用户创建自己的模板
